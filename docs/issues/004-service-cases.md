# 实现 Service Cases 列表和 Business Case 详情模板

Label: `ready-for-agent`

## What to build

Build the Service Cases experience, including a homepage case entry, case listing page, and Business Case detail template. Each Business Case should tell a structured story with client background, related service line, challenge, solution, execution content, and Case Result.

## Acceptance criteria

- [ ] Homepage includes a clear entry point to Service Cases.
- [ ] Service Cases list page displays available cases with useful summaries.
- [ ] Business Case detail pages include client background, related service line, challenge, solution, execution content, and Case Result.
- [ ] Case Result supports public metrics when available and qualitative impact when metrics are confidential.
- [ ] Case pages include Consultation CTA.
- [ ] Case content is maintained through Static Content Source data.
- [ ] Missing or confidential metrics do not produce empty result sections.
